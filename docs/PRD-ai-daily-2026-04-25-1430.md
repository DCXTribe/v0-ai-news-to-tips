**Document Version:** 1.0
**Created:** April 25, 2026, 14:30 UTC
**Status:** v1 Shipped

**Stack:** Next.js 16, Supabase (Auth + Postgres), AI SDK 6, Vercel AI Gateway

## 1. Executive Summary

**AI Daily** is a B2B-style productivity web app that translates AI news and updates into copy-paste-ready, role-specific tips that knowledge workers can apply in their daily work. It bridges the gap between "AI headlines" and "what do I actually do with this on Monday morning?"

The app supports the most widely used AI assistants — **Microsoft Copilot (Free & Pro/M365), ChatGPT (Free & Plus), Google Gemini (Free & Advanced), Claude (Free & Pro), and Perplexity (Free & Pro)** — and tailors every tip to the user's job role and the specific tools they have access to.

### Core Value Proposition

> "Stop reading AI news. Start using AI."

Most workers see AI headlines but don't translate them into action. AI Daily delivers a daily set of practical tips — each with a ready-to-paste prompt, a real workplace scenario, and a before/after comparison showing time saved.

## 2. Problem Statement

### The Gap
- AI tools ship new features weekly, but adoption among non-technical workers is shallow.
- News articles describe *what* AI can do, not *how to use it on a specific task this morning*.
- Free vs. paid tier confusion: users don't know what their tools can actually do.
- Generic prompt libraries don't account for role context (a marketer's needs differ from a finance manager's).

### The User Pain
- "I see Copilot got a new feature but I don't know if I have it or how to use it."
- "I'd like to use ChatGPT more at work but I don't know what to type."
- "I tried a prompt I saw on LinkedIn and it didn't fit what I do."

### The Opportunity
A focused, daily, role-aware utility that turns AI capability awareness into immediate, applied workflow change.

## 3. Goals & Success Metrics

### Primary Goals (v1)
1. Deliver at least **5 high-quality, copy-paste-ready tips per day** automatically.
2. Allow any visitor to translate an AI article or question into role-relevant tips in under **30 seconds**.
3. Give signed-up users a personal library of tips they have saved and tried.

### Success Metrics
| Metric | Target (90 days post-launch) |
| --- | --- |
| Daily active visitors | 1,000 |
| Sign-up conversion (visit → account) | 8% |
| Onboarding completion rate | 70% of signups |
| Tips saved per active user / week | 3+ |
| "Tried it" rate on saved tips | 40% |
| Translate / Ask actions per session | 1.5+ |

### Non-Goals (v1)
- Real-time RSS scraping of AI blogs (deferred to v2).
- Team / multi-seat workspaces.
- Mobile native apps.
- Browser extensions.
- Public sharing / social feed.
- Paid subscription tier.

## 4. Target Users

### Primary Persona: "AI-Curious Knowledge Worker"
- Marketers, salespeople, managers, HR, finance, ops, teachers.
- Has access to at least one AI tool (often through their employer).
- Uses AI occasionally (1-3x per week) but feels they could get more value.
- Reads AI news headlines but rarely acts on them.
- Wants concrete, fast wins — not theory.

### Secondary Persona: "AI Power User Looking for Inspiration"
- Already uses AI daily.
- Wants new prompts and patterns to expand repertoire.
- May use the Library feature to build a personal prompt collection.

### Out of Scope for v1
- Developers building AI products (different need: APIs, not prompts).
- Children / consumer leisure use cases.

---

## 5. Product Principles

1. **Practical over impressive.** Every tip must answer "what do I paste, where, and what happens?"
2. **Role-aware by default.** A finance manager and a teacher should see different tips for the same news item.
3. **Tool-honest.** Never suggest a feature the user's tier doesn't have. Tag every tip with the tools it works on.
4. **Fast to value.** Logged-out visitors see tips on the homepage immediately. No gating before value is delivered.
5. **Calm, professional UI.** This is a Monday-morning utility, not a hype machine.

## 6. Feature Scope (v1)

### 6.1 Today's Feed (Homepage `/`)

**Description:** Auto-generated daily collection of 5-6 AI news items, each translated into a tip card.

**Behavior:**
- First visitor of the day triggers AI generation (~15-30s).
- Result is cached in `ai_daily_feed` table for 24 hours.
- Subsequent visitors load instantly from cache.
- Public — no auth required to read.

**Tip Card Contents:**
- Headline news item (1 line).
- 1-paragraph "what's new" summary.
- Title of the actionable tip.
- Tools the tip works with (badge tags).
- Roles the tip is most useful for (badge tags).
- Estimated time saved.

### 6.2 Translate an Article (`/translate`)

**Description:** Paste a URL or full article text → receive 3-5 personalized tips.

**Inputs:**
- URL (fetched server-side and parsed) **OR** raw article text.
- Optional override of role / tools (defaults pulled from user profile if logged in).

**Output:**
- Tip cards rendered in the same format as the feed.
- If logged in, results are saved to `ai_daily_history`.

**Public — no auth required.**

### 6.3 Ask a Question (`/ask`)

**Description:** Free-form question input → AI returns relevant tips.

**Examples shown as clickable chips:**
- "How can I use Copilot in Excel as a finance manager?"
- "What's the best way to summarize meetings in ChatGPT Free?"
- "How do I use Gemini for lesson planning?"

**Output:** Same tip card format. Saved to history when logged in.

**Public — no auth required.**

### 6.4 Onboarding (`/onboarding`)

**Description:** 3-step setup that personalizes the entire app.

**Steps:**
1. **Role selection** — Marketer, Salesperson, Manager, Developer, Teacher, HR, Finance, Operations, Customer Support, Other.
2. **AI tools** — Multi-select of: Copilot Free, Copilot Pro/M365, ChatGPT Free, ChatGPT Plus, Gemini Free, Gemini Advanced, Claude, Claude Pro, Perplexity, Perplexity Pro.
3. **Skill level** — Beginner / Intermediate / Power User.

Stored in `ai_daily_profiles` and used by all generation prompts.

### 6.5 Library (`/library`, auth-protected)

**Description:** Personal collection of saved tips and translation history.

**Tabs:**
- **Saved tips** — Tips the user clicked "Save". Click to view detail, "Tried it" toggle, remove.
- **History** — Reverse-chronological log of articles translated and questions asked, each linking back to the generated tips.

### 6.6 Tip Detail (`/tips/[id]`)

**Description:** Permalink for a single tip — supports sharing and deep linking from history.

**Contents:**
- Full tip metadata (title, summary, scenario, before/after, prompt, tags).
- Save / Tried-it controls when authenticated.
- Copy-prompt button.

### 6.7 Authentication

- Email + password via Supabase Auth.
- Email confirmation flow with `/auth/sign-up-success` interim page.
- Auto-creation of `ai_daily_profiles` row on signup via Postgres trigger.
- Session refresh handled in `middleware.ts` via `lib/supabase/proxy.ts`.

## 7. Tip Anatomy (Editorial Spec)

Every tip the AI generates **must** include:

| Field | Description | Example |
| --- | --- | --- |
| `news_headline` | The short news item that motivated the tip. | "ChatGPT now supports persistent memory across chats" |
| `summary` | 1-2 sentence plain-English explanation. | "ChatGPT can now remember your role and preferences between sessions, so you don't have to re-explain context every time." |
| `tip_title` | Action-oriented title. | "Tell ChatGPT once who you are, save 5 minutes per chat" |
| `scenario` | Real workplace situation. | "You're a Marketing Manager who writes 10 emails per week. Each time you have to explain your tone and audience." |
| `before` | Manual / pre-AI workflow. | "Re-explain audience and tone in every chat. ~5 min lost per email." |
| `after` | AI-assisted workflow with the new tip. | "Save audience profile to memory once. Future drafts arrive in your tone instantly." |
| `prompt` | Copy-paste-ready, with `[PLACEHOLDERS]`. | `Remember: I'm a Marketing Manager at [COMPANY]. Our audience is [AUDIENCE]. Our tone is [TONE]. Apply these to all future drafts.` |
| `tools` | Array of tool slugs the tip works on. | `["chatgpt_plus", "chatgpt_free"]` |
| `roles` | Array of relevant role slugs. | `["marketing", "sales"]` |
| `time_saved_minutes` | Realistic estimate. | `5` |

### Editorial Rules (enforced in the system prompt)
- Copy-paste prompts must use `[BRACKETED]` placeholders for any user-specific input.
- No jargon ("LLM", "context window", "RAG") in user-facing copy.
- Never recommend a feature that requires a paid tier the user does not own.
- Time savings must be realistic (1-30 minutes), not "10x productivity."
- Scenarios must be concrete and grounded in everyday office tasks.

## 8. Technical Architecture

### 8.1 Stack
- **Framework:** Next.js 16 (App Router, RSC, Server Actions)
- **Language:** TypeScript
- **Styling:** Tailwind CSS v4 + shadcn/ui
- **Auth & DB:** Supabase (Postgres + Auth + RLS)
- **AI:** AI SDK 6 with default Vercel AI Gateway, model `openai/gpt-5-mini`
- **Hosting:** Vercel

### 8.2 Database Schema

All tables namespaced with `ai_daily_` prefix. Existing tables in the project (unrelated tax app) are untouched.

| Table | Purpose | RLS |
| --- | --- | --- |
| `ai_daily_profiles` | One row per user — role, tools, skill level. FK to `auth.users`. | User can only read/write own row. |
| `ai_daily_feed` | One row per day — cached generated feed. | Public read. |
| `ai_daily_tips` | All tips ever generated (feed, translate, ask). | Public read for `source = 'feed'`; owner-only otherwise. |
| `ai_daily_saves` | Join table: user ↔ saved tip, "tried it" flag. | User-owned. |
| `ai_daily_history` | User's translate/ask requests. | User-owned. |

A trigger `on_auth_user_created` auto-creates a profile row on signup.

### 8.3 API Routes

| Route | Method | Purpose |
| --- | --- | --- |
| `/api/translate` | POST | Generate tips from URL/text |
| `/api/ask` | POST | Generate tips from a question |
| `/api/save` | POST / DELETE | Save / unsave a tip, toggle "tried it" |

The Today's Feed page generates server-side directly via the helper in `lib/ai/generate.ts` and caches into `ai_daily_feed`.

### 8.4 AI Generation Layer (`lib/ai/generate.ts`)
- Uses `generateText` + `Output.object()` with a Zod-style schema enforcing the Tip Anatomy.
- Three prompt variants: `generateDailyFeed`, `translateFromArticle`, `askFromQuestion`.
- Single source-of-truth system prompt with editorial rules.
- All schema fields are required (`nullable` instead of `optional` for OpenAI strict mode compatibility).

### 8.5 Auth Flow
- `/auth/sign-up` → email confirmation → `/auth/sign-up-success`.
- Email link → `/auth/callback?code=...` → `exchangeCodeForSession` → redirected into app.
- `redirectTo` uses `NEXT_PUBLIC_DEV_SUPABASE_REDIRECT_URL` for v0 preview support.

### 8.6 Personalization Pipeline
1. User completes onboarding → `ai_daily_profiles` updated.
2. On every Translate / Ask / Tip-detail request, the route reads the user's profile.
3. Role + tools are injected into the AI system prompt.
4. Tools that the user does not own are explicitly excluded from suggestions.

## 10. User Journeys

### Journey A: First-Time Anonymous Visitor
1. Lands on `/`, sees today's 5-6 tips immediately.
2. Reads one, clicks Copy on the prompt.
3. Tries it in their AI tool — gets value.
4. Returns later in the day, scrolls more tips.
5. Hits "Save" → prompted to sign up.
6. Signs up → onboarding → returns to homepage with personalized banner.

### Journey B: Returning Power User
1. Logs in, lands on personalized homepage.
2. Goes to `/translate`, pastes an article from a Slack DM.
3. Gets 4 tips tailored to "Manager" + "Copilot Pro".
4. Saves 2 of them to library.
5. Marks one "tried it" the next day.

### Journey C: Question-Driven User
1. Has a specific task in mind ("summarize a 30-page PDF").
2. Goes to `/ask`, types the question.
3. Gets 3 tips comparing approaches in their available tools.
4. Picks the one that matches their tier, copies prompt.

## 11. Risks & Mitigations

| Risk | Mitigation |
| --- | --- |
| AI generates inaccurate tier claims (e.g. attributes a Pro feature to Free). | Strong editorial rules in system prompt + tool-tag enforcement; manual review feed in v1.5. |
| Cold-start latency on first daily visitor. | 24h caching; v1.5 will move to scheduled cron at 6am UTC. |
| Generic / repetitive tips over time. | Track recently-used news topics in `ai_daily_feed`, prompt to avoid repeats. |
| Article scraping fails on paywalls. | Fall back to user-pasted text input. |
| RLS misconfiguration leaks user history. | Schema reviewed; all user-owned tables have `auth.uid() = user_id` policies. |
| AI Gateway rate limit. | Cache feed; rate-limit per-IP on `/api/ask` and `/api/translate` (planned v1.1). |
