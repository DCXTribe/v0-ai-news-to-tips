## 1. Executive summary

AI Daily turns the firehose of AI product news into **practical, copy-paste tips that anyone can use today** in the AI tools they already have access to (Microsoft Copilot Free/Pro, ChatGPT Free/Plus, Google Gemini Free/Advanced, Anthropic Claude Free/Pro, Perplexity Free/Pro).

Every morning, an automated grounded pipeline reads the official OpenAI, Microsoft, Google, Anthropic, and Perplexity blogs, extracts what's new, and translates each item into a one-paragraph scenario, a copy-paste prompt, a "before & after" comparison, and a time-saved estimate — all citing the original source.

The product is a Next.js 16 web app backed by Supabase (auth + Postgres + RLS), with AI generation through the Vercel AI Gateway and grounded ingestion via Firecrawl + Tavily. It is mobile-first, sign-up-optional for browsing, and account-gated for saving and personalisation.

**One-line pitch:** *AI news, unpacked into things you can actually do today.*

## 2. Business context & opportunity

### 2.1 The problem

- AI capability is shipping faster than people can absorb it. Every week, OpenAI, Microsoft, Google, Anthropic, and Perplexity announce features that **could** save someone hours — but the gap between "I read the headline" and "I used it on Monday morning" is enormous.
- Most knowledge workers see only the surface (a headline, a screenshot, a tweet). They don't translate it into their own workflow.
- Generic prompt libraries (e.g. Awesome Prompts) are role-agnostic and disconnected from current news. Vendor blogs are dense and product-marketing-led.
- Internal "AI champions" inside companies spend hours each week curating tips for their teams. This is repetitive, manual work.

### 2.2 The opportunity

A **daily, grounded, role-tailored, copy-paste-ready** tip stream — sourced from official vendor blogs and translated by an LLM into role-specific actions — closes that gap in a way no existing product does.

### 2.3 Target outcome

- Users walk away every morning with **at least one new thing they can do today** in the AI tools they already have.
- Over time, their saved Library becomes a personal "AI playbook" — searchable, time-stamped, with a clear total of "time saved per use."
- For organisations, this becomes the cheapest path to AI adoption: distribute one URL to the whole team, no licences, no rollout plan.

## 3. Goals and success metrics

### 3.1 Product goals (v1)

| Goal | Definition of done |
|---|---|
| Make AI news immediately usable | Every published tip has a copy-paste prompt with bracketed placeholders, a real-world scenario, and a before/after comparison |

| Build trust through grounding | Every tip cites its source URL with publisher and publish date; up to 3 verbatim quotes; "Source-verified" badge on high-confidence tips |
| Build retention through Library | Users can save tips, browse history, and see a running total of time saved per use across their saved tips |
| Ship a friendly, mobile-first UX | Coral primary, sage success, warm cream surfaces, Inter typography, mobile bottom nav, single-column tip grids on phones |

| Metric | Target |
|---|---|
| Daily Active Users / Monthly Active Users | ≥ 25% (sign of habit formation) |
| % of authed users with ≥ 1 saved tip in first session | ≥ 40% |
| Average tips saved per active user per month | ≥ 6 |
| % of generations (Unpack/Ask) where source citation is verified | ≥ 95% |

## 5. Functional requirements (current state)

### 5.1 Today's feed (`/`)
- Public, no login required.
- Reads cached feed from Supabase (1-hour fallback revalidate, tag-based cache invalidation when cron writes).
- Displays the day's edition badge (e.g. "Saturday, April 25 edition") and 3-6 tip cards in a responsive grid (1 col mobile, 2 cols tablet, 3 cols desktop).
- Falls back gracefully to the most recent edition if today's hasn't been generated yet.
- Empty state explains the system and offers Unpack / Ask as alternatives.

### 5.2 Unpack (`/unpack`)
- Public, optionally authed.
- Accepts either a URL or pasted article text.
- URL path: scrapes via Firecrawl → returns clean markdown, title, publisher, publish date.
- Generates 3-5 grounded tips, each with the full tip anatomy (scenario, prompt, before/after, tools, time saved, source, citations, confidence).
- Anonymous: results show, plus a "don't lose this work — sign up" conversion CTA.
- Authed: results are persisted to `ai_daily_history` with `kind='paste'` and `tip_ids[]` linked into `ai_daily_tips`; "saved to your library" confirmation shown.

### 5.3 Ask (`/ask`)
- Public, optionally authed.
- Accepts a free-form question (e.g. "How can I use the new ChatGPT memory feature as a marketer?").
- Calls Tavily search to gather fresh, web-grounded context (last 30 days).
- Generates 3-5 grounded tips citing the search results.
- Same anonymous/authed treatment as Unpack.

### 5.4 Library (`/library`)
- Auth-gated.
- Two tabs: **Saved** and **History**.
- Stat strip at the top:
  - **Time saved per use** (sage / success accent) — sum of `time_saved` across saved tips, parsed from natural-language strings.
  - **If you use weekly** (sage) — projected monthly savings.
  - **Saved tips** — count.
  - **Top tool** — most-used AI tool across saved tips.
- Saved tab: full TipCards rendered for each saved tip.
- History tab: clickable list of all generations (Unpack + Ask), opens `/library/history/[id]` with the full set of tips re-rendered (scenario, prompt, before/after).
- "Your toolkit" chip row breaks down which AI tools your saved tips use.

### 5.5 Tip Card component (shared across Today / Unpack / Ask / Library)
- Meta strip: category chip, "Source-verified" / "May be outdated" badges, "Saves 30 min" pill.
- Headline + summary.
- Tool tags (e.g. "Copilot Pro", "ChatGPT Plus").
- Scenario card ("The scenario") with icon tile.
- Before & after section: muted dashed-border "The old way" panel, sage arrow connector, sage success "With AI" panel with thicker accent bar.
- Dark inverse-surface prompt block with one-tap copy.
- Source panel: publisher, publish date, clickable URL, up to 3 verbatim citations.
- Action bar: Copy + Save buttons (Save is auth-gated).

### 5.6 Authentication & onboarding
- Supabase Auth (email + password).
- `/auth/login`, `/auth/sign-up`, `/auth/sign-up-success`, `/auth/error`, `/auth/callback`.
- `useSearchParams` wrapped in `<Suspense>` (production-build-safe).
- `?next=` redirect parameter preserves the originating page across login.
- `/onboarding` captures role (Marketer, Manager, Sales, Dev, Teacher, HR, Finance, Ops, etc.), AI tools owned, skill level. Stored in `ai_daily_profiles`.
- Auto-create profile on signup via `auth.users` trigger.

### 5.7 Daily news pipeline (cron)
- `/api/cron/daily-feed`, scheduled via `vercel.json` for 06:00 UTC daily.
- Authenticated via either `Authorization: Bearer ${CRON_SECRET}` header (Vercel Cron) or `?secret=` query param (manual trigger).
- For each registered news source (currently 5 active: OpenAI, Microsoft Copilot, Google Workspace Updates, Anthropic, Perplexity):
  1. Tavily search (domain-restricted, last 14 days) → most recent relevant article URL.
  2. Firecrawl scrape that URL → clean markdown + metadata.
  3. AI SDK 6 `generateText` + `Output.object()` with `openai/gpt-5-mini` → grounded tip.
  4. Citation URL whitelist guard (any URL not equal to the canonical scraped URL is post-stripped).
  5. Write feed item + tip + source linkage to Supabase.
- Sources processed in **parallel** via `Promise.allSettled` (15-20s total, well under the 60s Hobby plan cap).
- After successful writes, calls `revalidateTag('daily-feed')` so the homepage cache refreshes immediately.
- Housekeeping: marks tips older than 30 days as `is_stale = true`.

## 6. Non-functional requirements

### 6.1 Performance
- Homepage feed served from cache (`unstable_cache`, tag `daily-feed`, 1-hour fallback revalidate).
- Cron parallelised across all sources.
- Article markdown trimmed to 8k chars before LLM call (sufficient for `gpt-5-mini`, ~30% latency reduction vs. 10k).
- All Tip Card content rendered server-side; no client-side data fetching for the feed.

### 6.2 Reliability
- Hobby plan max function duration is 60s. Cron's `maxDuration = 60`, `SOURCE_LIMIT = 5`. Sources skipped today are picked up tomorrow.
- Citation guard (URL whitelist) defends against hallucinated sources.
- Stale-tip housekeeping prevents the homepage from showing >30-day-old content as if fresh.
- RLS policies on all user-owned tables: `ai_daily_profiles`, `ai_daily_history`, `ai_daily_saves`, and user-generated rows in `ai_daily_tips`.

### 6.3 Security & privacy
- All custom tables namespaced `ai_daily_*` to avoid collision with the existing tax app schema in the same Supabase project.
- Row-Level Security on all user-owned tables. Public can read curated feed/tips only.
- `SUPABASE_SERVICE_ROLE_KEY` used only by cron (server-only, never exposed to client).
- `CRON_SECRET` gates manual cron triggers.
- Citations are URL-whitelisted server-side after generation.
- No PII in tips, history, or saves beyond what the user pastes themselves.

### 6.4 Accessibility
- Semantic landmarks (`<main>`, `<header>`, `<footer>`).
- ARIA labels on all icon-only buttons.
- Focus rings via design-token shadows.
- All copy contrasts ≥ AA against the warm cream background.
- Mobile bottom nav has a top-anchored active indicator dot for users who can't see the colour change.

### 6.5 Mobile-first responsiveness
- Single-column tip grids on phones, 2 cols at `md`, 3 cols at `xl`.
- Hero CTAs full-width on mobile, inline on `sm+`.
- Mobile bottom nav (Today / Unpack / Ask / Library) with safe-area inset for notched devices.
- All copy paths (Copy prompt, Save, Sign up) are thumb-reachable.

### 7.1 Data model (namespaced `ai_daily_*`)

| Table | Purpose | RLS |
|---|---|---|
| `ai_daily_profiles` | User role, tools, skill level | Owner-only |
| `ai_daily_news_sources` | Registry of 5 active vendor blogs | Service-role write, public read |
| `ai_daily_feed` | One row per published news+tip pair, per day | Public read for curated feed; tied to `source_id` |
| `ai_daily_tips` | The actual tip object: scenario, prompt, before/after, tools, time_saved, citations, source, confidence, is_stale | Public read for curated tips; user-owned tips visible only to owner |
| `ai_daily_saves` | Many-to-many: user × tip | Owner-only |
| `ai_daily_history` | Each Unpack / Ask generation: input, summary, kind, tip_ids[] | Owner-only |

### 7.2 Stack choices

- **Next.js 16** — App Router, Turbopack default, React 19.2, native cache APIs.
- **Supabase** — best fit for fast auth + Postgres + RLS in one integration.
- **AI SDK 6** with **Vercel AI Gateway** (`openai/gpt-5-mini`) — zero-config provider, structured output via `Output.object()`, future provider-portable.
- **Firecrawl** — clean web scraping that returns markdown + metadata reliably.
- **Tavily** — search API tuned for LLM grounding (recency filter, domain restriction, score-ranked results).

## 9. Client journey (end-to-end)

### Stage 1 — Discovery (Day 0, anonymous)

**Trigger:** Maya sees a colleague share `aidaily.app` in Slack.

1. Lands on `/` (Today's feed).
2. Sees the Saturday edition badge, the headline ("AI news, unpacked into things you can actually do today"), and a 3-up grid of today's tips.
3. Skims the meta-strip pills: "Saves 30 min · Productivity · Source-verified".
4. Clicks the first tip card — sees scenario, the dark prompt block, before & after, source citation.
5. **Aha moment:** "I can paste this prompt into Copilot at work right now."
6. Hits the Copy button. Toast confirms.

**Outcome:** Maya leaves with a usable prompt, no friction, no signup. The brand promise is delivered in under 60 seconds.

### Stage 2 — First active use (Day 0-1, anonymous)

**Trigger:** Maya wants to try it on her own news.

1. Reads the OpenAI blog about a new GPT-5 feature. Wants to know how to use it.
2. Clicks **Unpack** in the nav.
3. Pastes the OpenAI URL. Hits Unpack.
4. ~10 seconds later, sees 3-5 tips written for her role (Marketing) and her tools.
5. Sees the conversion CTA at the top of the results: *"Don't lose this work. Sign up free and we'll save these tips to your Library."*

**Outcome:** Strong signup intent. The CTA is contextual, not nagging — it appears after value is delivered, not before.

### Stage 3 — Activation (Day 1, signup + onboarding)

1. Maya clicks **Sign up free**, lands on `/auth/sign-up?next=/unpack`.
2. Enters email + password, confirms email via magic link.
3. Redirected to `/onboarding`:
   - Picks role: Marketing.
   - Picks tools: Microsoft Copilot Pro, ChatGPT Free.
   - Picks skill level: Beginner.
4. Lands back on `/unpack` with a "saved to your library" confirmation.
5. Now sees a Save button on every tip — taps Save on two of them.

**Outcome:** Profile created, history persisted, two tips in Library.

### Stage 4 — Habit formation (Days 2-30)

**Trigger:** Daily 09:00 routine — coffee, email, Slack, AI Daily.

1. Opens `/` on her phone during commute. Single-column grid, mobile bottom nav.
2. Skims today's 5 tips in 2 minutes. Saves the relevant one.
3. At the office, opens Copilot in Word. Pastes a saved prompt. **Saves 25 minutes** drafting a campaign brief.
4. After 2 weeks, opens **Library**:
   - **Time saved per use: 4h 35m.**
   - **If you use weekly: ~18h/mo.**
   - **Saved tips: 12.**
   - **Top tool: Copilot Pro.**

**Outcome:** Maya has a quantified personal AI playbook. The "time saved" total is the moment that converts her from user to advocate.

### Stage 5 — Advocacy (Day 30+)

1. Maya forwards her Library URL to a colleague: "look at how much this saved me."
2. Asks her IT lead (David) to share AI Daily on the team Slack.
3. David subscribes himself, finds the source-citations layer makes the feed forwardable to the whole team without him having to fact-check anything.

**Outcome:** Organic, low-friction word-of-mouth growth. The Library's quantified time-saved is the shareable artefact.

### Stage 6 — Long-term retention

- Daily cron means there's always something new.
- Saved Library is sticky — the longer they use it, the more time-saved accumulates, the harder it is to leave.
- Stale-tip badge keeps the source-of-truth honest as features evolve.
- Future: weekly digest email, team Library, browser extension.

## 10. Constraints, risks & dependencies

### 10.1 Hard constraints
- Vercel Hobby plan: max function duration 60s, max cron resolution 1/day for unauthenticated jobs.
- Free-tier API quotas on Firecrawl and Tavily (need to monitor; both have generous starter limits).
- Supabase free tier: 500 MB DB, 50k MAU.
- AI Gateway: zero-config for OpenAI provider, but tokens still meter — `gpt-5-mini` chosen specifically for cost.

### 10.2 Risks & mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Vendor blog scrape breaks (selector changes) | Medium | Medium | Tavily-first source discovery is selector-free; Firecrawl handles markup variance |
| LLM hallucinates citations | Medium | High | Server-side URL whitelist post-strips any non-canonical citations |
| Cron timeout under load | Low | Medium | Parallel sources, 60s cap, `Promise.allSettled` so one bad source doesn't kill the run |
| Privacy concern from pasted articles | Low | High | We only persist input + summary if user is authed; anonymous Unpack/Ask is fully ephemeral |
| Stale tips appear as fresh | Low | Medium | Daily housekeeping marks `is_stale` on tips >30 days |
| API costs balloon | Low | Medium | `gpt-5-mini`, 8k char input cap, public cache, source limit on cron |

### 10.3 Dependencies
- Vercel (hosting + cron + AI Gateway).
- Supabase (auth + DB).
- Firecrawl (web scraping).
- Tavily (web search).
- OpenAI (via AI Gateway, model `gpt-5-mini`).
